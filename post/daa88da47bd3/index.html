使用Fiddler抓包手机请求记录

一次使用fiddler抓包手机请求的流水账记录

已有条件

  • Fiddler - 版本忘了
  • 手机 - Redmi Note 10 Pro MIUI 13 Android 12
  • 网络环境 - 笔记本 和 手机 链接同一Wifi

软件设置

设置Https

标题栏选择 Tools -> Options… -> HTTPS

注意对比图中参数，或根据文本勾选以下三种选项

  • Capture HTTPS CONNECTs
  • Decrypt HTTPS traffic
  • Ignore server certificate errors (unsafe)

配置证书

选中Decrpt HTTPS traffic 后 Fiddler就可以截获HTTPS请求，第一次选择会弹出证书安装提示，若没有弹出提示，勾选Actions-> Trust Root Certificate 一路Yes以配置证书

这里面配置证书可能会出现两种提示框，注意出现如CN = DO_NOT … FiddlerRoot 或 (CA)安装证书: DO_NOT … FiddlerRoot的文本内容提示，这里面DO_NOT … FiddlerRoot就是安装的证书名称

确认是否安装证书可以通过 Actions -> Open Windows Certificate Manager -> 受信任的根证书颁发机构 -> 证书 -> 右侧列表中寻找是否有上面配置证书时出现的证书名称

设置过滤请求

当前过滤设置为默认的from all processes是获取，后面步骤为手机设置代理抓取请求后，fiddler会抓到当前设备和手机的所有请求。这里开启过滤功能来抓取局域网内android手机的请求包，设置 from remote clients only 来只获取远程客户端即手机的请求

设置软件允许远程连接

标题栏下 Tools -> Connections -> 勾选Allow remote computers to connect 即可

重启并验证是否启用

设置完成后重启软件使得配置生效，然后使用快捷键win + r打开cmd命令行输入以下内容

注意下面查找的端口8888是对应上面Connection选项卡中的Fiddler Classiclistens onport输入框中的内容

输入命令后查看第一行发现本地ip 8888 端口的状态为LISTENING (即监听中)

手机移动端设置

检查设备位于同一局域网

首先确保手机和电脑在同一局域网内，查看相应网络设备的ipv4地址，一般是根据路由器的IP地址分配，如路由器网关为192.168.0.1(大部分情况下这个也是路由器网络配置面板),则设备链接路由器网络(如wifi或直连网线)时设备ip也被分配在相同网段内

打开命令行输入 ipconfig来获取ipv4地址或将鼠标移动至Fiddler软件标题栏下右侧的Online选项上获取ipv4地址

手机上链接与电脑同一网络后，两个设备就在同一局域网内。注意关闭流量选项避免要监听的请求偷跑流量请求

设置代理

手机打开当前链接wifi设置/高级设置 设置开启手动代理

配置主机名为电脑设备ip地址，端口为上面默认配置的8888

然后在浏览器中输入 主机名:端口 ，我这里是192.168.0.161:8888 来链接Fiddler服务获取证书

点击FiddlerRoot certificate 下载链接下载证书

打开手机设置 选择 安全 -> 更多设置 -> 加密与凭据

点击安装证书 -> CA证书 选中下载的Fiddler Root.crt文件 安装凭证

安装后退回 加密与凭据 ，选择信任的凭据 选择用户选项卡后查看是否出现 DO_NOT_TRUST_BC 之类出现FiddlerRoot字样的 相关证书确认安装证书成功

需要注意不同设备上的Fiddler提供的证书都是不一样的，测试完抓包后记得删除，因为当前配置证书值对应当前电脑设备上的fiddler，后续使用其他设备上的fiddler链接抓包时无法使用该证书

后续手机网络请求都可以在Fiddler应用中查看抓包内容

抓包测试完毕后关闭代理

测试完毕后关闭代理避免代理开启影响网络环境从而导致日常网络使用受影响
